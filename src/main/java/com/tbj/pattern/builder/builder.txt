首先便是先前的坏处，应该有以下两点。
1、制造分身非常复杂，每次都要现在制造，容易丢三落四，去胳膊少腿的。
2、分身的样子是可以变化的，所以假设我要重复制造几个瘦瘦的家伙的话，那就不得不每次将整个建造者过程和身体的各个部分全部重来一遍了。

如果换成专业的语言来总结的话，则是下面两点。
1、对象的构建过程非常复杂，每次客户端调用的时候，都要自己控制构建的过程，这会导致构建过程的重复，而且非常容易忘记某一个步骤。
2、对象的表示是可以变化的，所以如果要重复制造几个表示相同对象的时候，只能每次都将过程和表示重复一遍。

那么现在改变了方式之后的好处则是下面两点。
1、分身制造的过程被灵魂专门记忆和控制，所以不会出现丢三落四的情况。
2、由于分身的样子由固定的分身制造者提供，所以如果要重复制造相同样子的分身，只需要让灵魂控制过程再重来一次即可。

而好处的话，如果换成专业的语言来总结的话，则是下面两点。
1、最主要的好处就是使对象的构建与表示分离，可以让表示独立过程而独立变化，并且客户端不在需要关心具体的构建过程和表示，
2、由于固定的表示由具体的制造者提供，所以要重复制造几个表示相同对象的时候，只需要让指挥官控制过程再制造一次即可。

“看来这个建造者模式很适合我制造分身的情况啊，那么总结下它适用的场景吧，它适用的场景应该是这样的。”

1、当对象的构建过程十分复杂，需要与表示分离的时候。
2、当对象的表示有不同的种类，需要加以区分的时候。
